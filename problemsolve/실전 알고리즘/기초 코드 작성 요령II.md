# 기초 코드 작성 요령 II

## STL과 함수 인자 (참조자 사용)

- 값에 의한 전달(Pass by Value)
  - 함수의 인자 `vector`, `string`등 STL 컨테이너를 그냥 넘겨주면, 데이터 전체가 복사되어 전달된다.
  - 비용: $O(N)$
  - 예시: `bool cmp(vector<int> v1, vector<int> v2)` → N이 크면 시간초과의 원인이 된다.
- 참조에 의한 전달(Pass by Reference)
  - 데이터 복사 없이 원본을 참조하므로 효율적이다.
  - 비용:  $O(1)$
  - 함수 내부에서 값을 바꾸지 않는다면 `const`를 붙여주는것이 좋다.
  - 예시: `bool cmp(const vector<int>& v1, const vector<int>& v2)`

## 표준 입출력
C++의 `cin, cout`은 편의성이 좋지만, 기본 설정 상태에서는 느리다. 코테에서의 입력이 많을때는 입출력 속도때문에 시간 초과 발생 가능성이 있다.

### 입출력 속도 최적화 코드
```cpp
int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    // 코드 구현
}
```
- `ios::sync_with_stdio(0)`: C stream과 C++ stream의 동기화를 끊는다. 이 코드를 사용한 이후에는 절대 `scanf, printf` 등을 `cin, cout`과 섞어 쓰지 않도록 한다.
- `cin.tie(0)`: `cin`과 `cout`의 묶음을 푼다. 기본적으로 c++에서 입력을 받기전에 출력 버퍼를 비우지만 이를 막아서 속도를 높이기 위함.

### 공백 문자열 처리
- `cin`과 `scanf` 모두 공백을 포함한 문자열을 받을 때 껄끄럽다. 둘 다 공백 앞까지만 입력을 받기 때문이다.
- 이를 해결하기 위해서는 그냥 `getline`을 사용하도록 하자.

### endl vs \n
`endl`은 매우 느리므로 사용하지말자. 그냥 `\n`으로 개행하면 된다.

## 코드 작성 관련 팁
코딩테스트는 정확하고 빠르게 돌아가는 코드를 작성하여 정답을 찾는것이 목표이지 실제 개발처럼 코드 스타일, 가독성, 클린코드를 고집할 필요가 없다. 내가 작성할 수 있는 최적의 방법으로 제일 빠르고 정확한 코드를 작성하자.
- 헤더파일: `bits/stdc++.h>`
- 코딩 스타일
  - 코드를 클린하게 짜는것보다 디버깅하기쉽고 일관된 스타일을 유지하자.
  - 출력 형식이 복잡하지 않으면 `cout`을 사용하고, 복잡하다면(소수점, 특정 포맷 등)에서는 `printf`를 쓰는 등 유연하게 대처하자. 동기화를 끊었을 때 두 방식을 섞지만 않으면 된다.
